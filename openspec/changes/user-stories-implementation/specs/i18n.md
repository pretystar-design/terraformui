# Spec: i18n (US-011)

## User Story
As a user, I want UI language toggling between Chinese and English so that the tool is accessible to a broader team.

## Requirements

### R1: Language Toggle
- Toolbar language switcher (EN / 中文)
- Instant UI update without page reload
- Language preference persisted in localStorage

### R2: Translation Coverage
- All UI labels, buttons, tooltips, messages
- Provider and resource names (kept in English for Terraform compatibility)
- Error messages and validation text
- Toolbar items, panel headers, empty states

### R3: Implementation
- i18next with JSON translation files
- Translation keys: `namespace.key` format
- Fallback to English for missing translations

### R4: RTL Support
- Not required (Chinese and English are both LTR)

## Acceptance Criteria
- Clicking language toggle instantly switches all UI text
- Reloading page preserves selected language
- All toolbar items, panel headers, buttons translated
- Missing keys fall back to English without errors
- Resource type names remain in English (e.g., `aws_instance`)
